Projects

BostelCRM utilizes Cases to document customer interactions where they seek assistance or guidance, typically in sales or support functions. Users can create, update, assign, and close Cases as they work on them, while tracking incoming and outgoing communication threads at each stage. This ensures a comprehensive record of the Case's progress is maintained within the CRM. Additionally, Cases can be associated with specific records, such as Contacts.

Cases Actions

Users can access the Projects actions from the Projects module menu drop-down or via the Sidebar after clicking to view the module. The available Project actions are:

  • Create Project

    Clicking on the "Create" option opens a new form in the Edit View mode that enables users to create a new Project record.

  • View Projects

    Selecting the "List Projects" option redirects users to the List View for the Projects module, which enables them to search and display a list of Project records.

  • Resource Calendar

    This feature enables users to search for projects based on various parameters such as project name, users, contacts, type, and date.

  • View Projects Tasks

    The "List Project Tasks" option enables users to display a list of Project Tasks that are related to a parent Project.

Creating Projects

The Projects module allows you to create, manage, and duplicate both Projects and Project Tasks. For each Project, you can define multiple Project Tasks. To create a Project Task, you need to associate it with a Project. Additionally, a Project can be associated with multiple activities, Opportunities, and Cases.

Once you save the Project form, you will be redirected to the Project's detail page where you can view all the details related to the project. From this page, you can associate the project with other records such as Contacts and Opportunities.

Creating Projects Tasks

To create a new task in the Project Tasks sub-panel, you need to click on the "Add" button. Once on the Project Tasks page, you can fill in the required information for the task, including the name, task number, start date, finish date, percentage complete, priority level, milestone status, project name, and notes. You can then save the task by clicking the "Save" button or cancel the task creation by clicking the "Cancel" button. This process allows you to create and manage multiple tasks associated with a project in the Projects module.

Managing Projects & Project Tasks

  • Clicking on any sortable column title in the Cases List View allows you to sort records in ascending or descending order based on the selected column.
  • Refer to the Search section of the user guide to search for a specific Case.
  • To view the details of a Case, click on the Case Subject in the List View, which will open the record in Detail View.
  • To edit the Case details, either click the Edit icon in the List View or click the edit button on the Detail View under the Actions tab, make the required changes, and click Save.
  • To duplicate a Case, click the Duplicate button on the Detail View and then save the duplicate record.
  • To merge duplicate Cases, select the records from the Cases List View, click the Merge link in the Actions drop-down list, and follow the merge process.
  • To delete a Case, navigate to the Detail View and click the Delete button under the Actions tab.
  • To track changes made to audited fields, you can click the View Change Log button under the Actions tab in the Case record's Detail View or Edit View.
